# Career Motivations Reflection & Growth Direction Assessment App
> Module arc generated by Claude in June 2026 based on direction from Jared and exploration / research into the core resources listed below.

A self-directed career-coaching curriculum bakes into a Metalab Sites app to answer "what actually drives me, and where am I going?"
Structured as the standard multi-module arc used by professional coaches (ICF-aligned, with senior-IC / engineering-leader inflection),
adapted as worksheets to be filled in at the driving individual's pace.

> **Not a march, a menu.** Doesn't have to be completed sequentially, but some modules are dependent on others
(6 depends on 5; and 5 depends on some lightweight 360 peer feedback)

## How to use each module

Every module file has the same shape:

1. **Purpose** — the question this module answers
2. **Essential references** — what to take/read first (free vs. paid noted)
3. **Exercises** — concrete things to do
4. **Worksheet** — fill-in section; my answers live here


## Pacing
Your own pace. Expect a solid 1/2-full day's effort for several of the modules; so the whole course is likely to take a couple of weeks to work through if you're doing it off the side of your desk / out of work hours.
Set deadlines for yourself that work for you.
(Future feature: scheduled reminders)

## Technical Setup
- React via React-router (framework mode)
  - 🚀 SPA mode — `ssr: false` (static client build; no runtime server, since Metalab Sites is a static host)
  - ⚡️ Hot Module Replacement (HMR)
  - 📦 Asset bundling and optimization
  - 🔄 Data loading and mutations
  - 🔒 TypeScript by default
  - 🎉 TailwindCSS for styling
  - 📖 [React Router docs](https://reactrouter.com/)

- Metalab Sites (2.0) https://sites.metalab.com/docs
  - Deployment via the Sites CLI (GitHub branch auto-deploy can't target a build sub-folder — see Deployment below)
  - User Identities via Okta Auth
  - User-specific content stored in localStorage only, because Sites db doens't have user-specific read security, and the content of the worksheets is likely to be personal information.

### Local Development
`npm install` > `npm run dev` : `http://localhost:5173`

To preview the **production build** locally as a static site (mirrors how Sites serves it):
```bash
npm run build
npx serve build/client     # then open the printed localhost URL
```
> Note: open the **clean directory URL**, not `…/index.html`. `index.html` is a file, not a route — visiting it
> directly makes the SPA router match no route and render the 404 page (and a hydration mismatch in the prod build).


### Metalab Sites Deployment

Deploying via the **Sites CLI** (terminal publish) — *not* GitHub branch auto-deploy.

**Deploy:**
```bash
npm run build
sites deploy ./build/client career-motivations-worksheet
```
- We publish **`build/client`** (the static client output of React Router's SPA build), not the repo root and not `dist/`.
- Live at: `https://sites.metalab.com/jared/career-motivations-worksheet/`

**⚠️ Sub-path configuration (required — the app is served from a URL sub-path, so two separate settings must agree):**

| Concern | Tool | Setting | Value |
|---|---|---|---|
| Asset URLs in the built HTML | Vite | `base` (`vite.config.ts`) | `/jared/career-motivations-worksheet/` |
| Runtime route matching | React Router | `basename` (`react-router.config.ts`) | `/jared/career-motivations-worksheet/` |

Both must match the deployed sub-path (leading **and** trailing slash). If only `base` is set, assets load but the
router 404s; if neither, assets fail to load (browser gets `text/html` where it expects JS/CSS). See the 2026-06-16
debugging notes for the full diagnosis.

> **Why not GitHub auto-deploy?** Sites' GitHub integration extracts the **entire repo root** to the site
> directory, with no way to scope the deploy to a build sub-folder. The docs reference a "Source path" setting
> for exactly this, but per Jeremy (Sites maintainer, 2026-06-16) that's an **AI hallucination in the docs** —
> sub-directory support doesn't actually exist. So GitHub auto-deploy is unusable here (it'd ship the repo root,
> which just renders as a folder listing). GitHub connection has been left disconnected.
